A couple of changes I’d recommend before building:

* Don’t hardcode the super admin username and password into the application or repository. Instead, store them as environment variables or create the first super admin during deployment. This is much more secure.
* Rather than live location tracking at all times, consider periodic updates (for example, every few minutes while on duty or on important events). That reduces battery use, mobile data, and privacy concerns.
* Design the backend as a multi-tenant system so each post office’s data is logically isolated while still using one deployment. This will make scaling to thousands of offices much easier.
